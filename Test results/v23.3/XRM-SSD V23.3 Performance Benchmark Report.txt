XRM-SSD V23.3 Performance Benchmark Report
Platform: NVIDIA L4 GPU (Native Mode)
1. Executive Summary
This report presents the performance characteristics of XRM-SSD V23.3 running natively on an NVIDIA L4 GPU. The system was evaluated across batch sizes 1, 4, 8, and 16, measuring throughput (TPS), average latency, and P95 latency.

Key Finding: The optimal batch size for maximum throughput is 8, achieving 194.9 TPS with an average latency of 41.1 ms and P95 latency of 64.7 ms.

5. Analysis & Key Insights
5.1 Throughput Scaling
Batch 1 → 4: Throughput increases by +34.2% (135.9 → 182.5 TPS)
Batch 4 → 8: Throughput increases by +6.8% (182.5 → 194.9 TPS) – peak achieved
Batch 8 → 16: Throughput decreases by -6.9% (194.9 → 181.4 TPS)

The GPU reaches saturation at batch size 8. Larger batches increase latency without improving throughput, 
likely due to memory bandwidth constraints or kernel execution overhead.

5.2 Latency Characteristics
P95 Latency remains under 65 ms for batch sizes ≤8.
At batch size 16, P95 latency increases to 130.3 ms (2× batch 8), making it unsuitable for real-time applications.

5.3 Efficiency Observation
The system shows diminishing returns beyond batch size 8, suggesting optimal utilization of L4's Tensor Cores and memory subsystem at moderate batch sizes.

7. Conclusion
XRM-SSD V23.3 on NVIDIA L4 GPU delivers stable and predictable performance with a clear optimal operating point at batch size 8 (194.9 TPS, 64.7 ms P95 latency). 
For latency-sensitive applications, batch size 1 provides sub-8 ms P95 latency while still achieving 136 TPS.

No stability issues were observed across all tested configurations. The system is production-ready for both real-time and batch inference workloads.
